## Architecture

- `src/core/` defines shared config, database helpers, and Pydantic domain contracts, including conservative live-guardrail settings that stay inactive unless explicitly armed.
- `src/chain/` contains Solana/Jupiter/wallet integration placeholders plus a pre-live Jito block-engine scaffold for MEV-aware bundle request building and injectable submission testing.
- `src/risk/` contains token risk checks, aggregate scoring, funding and honeypot research helpers, signal-aware enrichment, RugCheck-assisted authority/holder/honeypot data, and parallelized provider prechecks. Strict checks retain `UNKNOWN` when evidence is absent; paper-only discovery relaxation is explicit `discovery_relaxed` research metadata, not strict approval. Jupiter quote output is not accepted as pool liquidity.
- `src/execution/` defines the execution adapter contract plus paper and live adapter implementations. Its DexScreener price provider accepts only requested-mint / wSOL base-oriented, finite positive SOL prices; unsupported, inverted, USDC-quoted, and non-finite data degrades to unavailable. Paper PnL preserves unavailable entry/mark states and uses remaining-token cost basis for partial exits. The rest of the execution layer includes disabled-by-default live submission helpers, guardrails, preflight, execution-config validation, reconciliation, circuit breaker, readiness, guarded live exit, and guarded live buy; live buy records unpriced fills until actual fill data exists.
