## Known Issues

- All three protocol-based providers are now implemented and healthy when env config is present: `transaction_simulator` (available), `wallet_balance_lookup` (provider reached, wallet unfunded — insufficient balance), `wallet_holdings_lookup` (available). Position reconciliation passes (`no_live_positions_to_reconcile`) — paper-mode positions no longer trigger false mismatches. The remaining blocker before a guarded micro-live smoke test is live arming config: `TRADING_WALLET_PRIVATE_KEY`, `LIVE_TRADING_ENABLED`, confirmation phrase, `LIVE_KILL_SWITCH`, tiny live caps, RPC URLs.
- Live Jupiter execution is still intentionally disabled for direct swaps; the serialized submission helper only wires optional Jito/RPC routing for future guarded use and does not enable trading by itself.
- Whale tracker is polling-only for now; webhook ingestion still needs a public callback endpoint and receiver.
- pump.fun websocket token-creation and migration subscriptions are now verified against PumpPortal, but the short dry-run did not capture a live migration payload; graduation normalization is still proven by offline fixtures rather than observed runtime traffic.
- Risk checks use conservative local token fields until on-chain enrichment is implemented.
- Dynamic exits are only active when `exits.dynamic_exits_enabled` is enabled and the exit runtime receives `DynamicExitState` context.
- Latest integrated paper-cycle runs still produce 0 strict paper trades, and MT-054 confirms holder concentration remains the dominant strict-mode blocker even after the narrow pump.fun bonding-curve artifact fix; the current recommendation is to treat further holder work as a parallel calibration lane while attention/narrative scoring becomes the primary lane for ranking discovery candidates that now make it through the warning-policy stack. MT-055 now makes capacity blockers explicit and adds a fresh-session paper-only evaluation mode when older persisted paper positions would otherwise mask that ranking signal.
- Liquidity-aware sizing remains disabled by default; operators must opt in through `position.liquidity_sizing_enabled` before the new conservative liquidity caps affect decisions.
